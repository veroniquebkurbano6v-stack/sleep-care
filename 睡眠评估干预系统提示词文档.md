# 睡眠评估干预系统设计提示文档

## MVP功能清单

本系统的最小可行产品（MVP）需实现不少于18项核心功能，覆盖智能感知、环境调控、用户交互与数字疗法四大模块。以下功能列表为后续系统开发提供明确的技术边界与实现路径。

### 智能感知模块

1. **毫米波生理参数监测**  
   【业务描述】通过非接触式毫米波雷达实时采集呼吸频率、心率、体动等生理信号，支持高精度睡眠分期识别。  
   【关键技术实现要点】采用60GHz或77GHz频段雷达，探测距离0.2–3米；呼吸误差≤±1次/分钟，心率误差≤±2次/分钟；通过FMCW调制与FFT分析提取生命体征。

2. **异常事件检测预警**  
   【业务描述】自动识别呼吸暂停、周期性肢体运动、频繁觉醒等异常事件并记录，用于风险评估与报告生成。  
   【关键技术实现要点】设定阈值模型（如连续10秒无呼吸波动判定为暂停），事件标签写入`events_json`字段，支持后期临床复核。

3. **环境多因子同步感知**  
   【业务描述】集成光、温、湿、声传感器，全面监测卧室微环境状态，构建环境画像。  
   【关键技术实现要点】配置环境光传感器（0–1000 lux，低照区精度±10%）、数字麦克风（SNR≥65 dBA）、温湿度传感器（±0.5°C/±3%RH），数据时间戳对齐。

4. **睡眠阶段自动识别**  
   【业务描述】融合生理信号动态变化，自动划分入睡、浅睡、深睡、REM及清醒阶段，生成睡眠结构曲线。  
   【关键技术实现要点】基于机器学习模型或规则引擎分析HRV、呼吸变异性、体动频次，输出每30秒一个编码的`stage_curve`序列（0=清醒,1=浅睡,2=深睡,3=REM）。

### 环境调控模块

5. **助眠光照自动调节**  
   【业务描述】根据就寝时间预设或手动启动，开启低色温、低亮度灯光，营造入睡氛围。  
   【关键技术实现要点】LED系统支持色温2200K–3000K、亮度≤30 lux稳定输出；采用PWM≥2000Hz或线性驱动，确保无闪烁。

6. **日落程序渐暗控制**  
   【业务描述】在就寝前执行“日落”模式，灯光缓慢降低，模拟自然黄昏过程。  
   【关键技术实现要点】亮度与色温同步线性下降，持续时间可调（5–30分钟，默认10分钟），变化平滑无跳变。

7. **夜间微光安全指引**  
   【业务描述】当检测到用户夜间离床活动时，自动开启极低亮度照明（≤10 lux），保障安全且不影响再次入睡。  
   【关键技术实现要点】结合雷达人体定位实现定向补光，LED最低支持1 lux恒定输出，避免强光刺激。

8. **清晨日出唤醒程序**  
   【业务描述】在预设起床时间前5–10分钟启动“日出”模式，灯光渐亮至≥250 lux、色温升至≥4000K，辅助自然唤醒。  
   【关键技术实现要点】触发方式为定时或结合睡眠阶段预测（如从深睡转入浅睡时提前启动），支持用户自定义时长。

9. **助眠音效智能播放**  
   【业务描述】提供白噪音、海浪、雨声等多种音频选择，支持根据呼吸节奏动态调节音律引导放松。  
   【关键技术实现要点】内置音频库本地存储，支持试听；未来可通过相位锁定实现生物反馈同步。

10. **主动降噪抗干扰**  
    【业务描述】监测突发噪音（如犬吠、关门声），启动反向声波抵消或播放掩蔽音效维持安静环境。  
    【关键技术实现要点】麦克风实时监听，触发后扬声器输出反相声波或切换至粉红噪声，抑制突发干扰。

### 用户交互模块

11. **多功能物理旋钮交互**  
    【业务描述】设备配备主按键，支持短按启动、双击切换模式、长按旋转调节参数，提供即时控制能力。  
    【关键技术实现要点】硬件集成编码器+按键，支持三种手势识别；状态指示灯配合颜色反馈当前模式。

12. **专用停止键应急控制**  
    【业务描述】设置独立物理按钮，可在任何音频播放状态下立即暂停/停止声音输出，提升安全感。  
    【关键技术实现要点】按键直连中断服务，优先级最高，确保零延迟响应。

13. **微信小程序全流程管理**  
    【业务描述】提供移动端统一入口，支持账号管理、设备绑定、睡眠报告查看、设置调整等功能。  
    【关键技术实现要点】基于微信原生框架开发，JWT认证机制保障安全，适配iOS/Android双平台。

14. **睡眠报告可视化展示**  
    【业务描述】自动生成每日睡眠报告，首页展示评分、时长、深睡占比等关键指标，支持图表深度浏览。  
    【关键技术实现要点】集成堆叠条形图（睡眠阶段）、折线图（心率/呼吸趋势）、事件列表，支持日/周/月视图切换。

15. **报告一键分享与导出**  
    【业务描述】支持将睡眠报告生成图片或PDF格式，通过微信分享给家人或医生，也可导出CSV用于专业分析。  
    【关键技术实现要点】后台生成可视化快照图像，支持水印添加；CSV导出不含PII信息，符合隐私保护要求。

### 数字疗法模块

16. **e-CBTi睡眠卫生教育**  
    【业务描述】提供图文形式的睡眠健康知识科普内容，帮助用户建立良好睡眠习惯认知。  
    【关键技术实现要点】内容由医学团队提供，小程序内分类展示，支持收藏与历史浏览记录。

17. **结构化睡眠日记填写**  
    【业务描述】用户每日填写主观睡眠日志，包括上床/起床时间、入睡潜伏期、觉醒次数、日间困倦程度等。  
    【关键技术实现要点】表单字段标准化，支持日历视图快速录入与回顾，数据用于e-CBTi治疗依从性评估。

18. **ISI/PSQI/ESS量表测评**  
    【业务描述】内置失眠严重程度指数（ISI）、匹兹堡睡眠质量指数量表（PSQI）、Epworth嗜睡量表（ESS），自动评分并分级。  
    【关键技术实现要点】提交后即时计算得分，依据临床标准自动评定等级（如ISI: 0–7=无显著失眠），结果存入`questionnaire_results`表。

上述功能构成了系统的核心能力集，其技术实现依赖于下述系统架构的支持。

## 系统架构说明

本系统采用前后端分离架构，主要由以下组件构成，支撑前述MVP功能的完整运行：

- **微信小程序（前端）**：作为用户直接交互界面，负责展示数据、接收操作指令，并通过HTTPS协议调用云后台API获取资源。
- **云后台API服务（后端）**：基于RESTful接口提供用户管理、设备管理、睡眠报告查询、e-CBTi数据处理等核心服务，部署于云端服务器，对外暴露统一接口。
- **数据库（MySQL）**：持久化存储用户信息、设备状态、睡眠报告、睡眠日记、量表结果等核心数据，支持高效查询与事务处理。
- **模拟数据生成器（后台任务）**：每日凌晨2点为绑定虚拟设备的用户生成符合生理规律的模拟睡眠数据，用于无硬件阶段的功能验证。
- **医生端Web后台**：提供独立认证入口，供医生查看授权患者的脱敏数据（报告、日记、测评结果），支持添加干预建议。
- **虚拟设备模拟器**：替代真实硬件上报心跳、状态，接收配置更新，用于测试设备绑定、在线状态等场景。
- **OTA固件管理模块**：预留接口，支持上传新版本固件、发布更新日志、标记强制升级，为后续硬件接入做准备。
- **消息推送服务**：基于微信订阅消息机制，在早晨推送“睡眠报告已生成”提醒，设备离线超24小时时发送告警。

各组件之间的数据流向如下：

1. 用户在微信小程序发起请求 → 调用云后台API → 查询数据库返回数据 → 小程序渲染展示；
2. 模拟数据生成器每日定时运行 → 生成睡眠数据 → 写入数据库sleep_reports表；
3. 医生登录Web后台 → 验证身份 → 查询doctor_authorizations表 → 获取患者列表 → 调用API获取脱敏报告数据；
4. OTA管理模块上传固件 → API暴露最新版本信息 → 小程序检查更新 → 用户确认安装 → 记录日志（硬件接入后触发真实升级）。

该架构设计支持“无硬件依赖开发”，待真实设备就绪后仅需替换数据源与控制指令下发模块即可无缝对接，具备良好的演进性与扩展能力。

## 数据库ER图规范说明

为支撑上述系统架构与功能实现，数据库需设计5张核心数据表，每张表均遵循规范化原则，确保数据一致性与查询效率。

### 表1：用户表 (users)

| 字段名 | 数据类型 | 主外键约束 | 非空规则 | 业务说明 |
|--------|----------|------------|-----------|---------|
| user_id | bigint | PRIMARY KEY, AUTO_INCREMENT | NOT NULL | 用户唯一标识，系统自动生成 |
| phone | varchar(20) | UNIQUE | NOT NULL | 手机号码，用于登录和身份识别 |
| nickname | varchar(50) | — | NOT NULL, DEFAULT '用户' | 用户昵称，最长50字符 |
| avatar_url | varchar(255) | — | NULL | 头像URL地址 |
| gender | tinyint | — | NOT NULL, DEFAULT 0 | 性别：0=未知，1=男，2=女 |
| birth_year | int | — | NULL | 出生年份，用于年龄计算 |
| role | tinyint | — | NOT NULL, DEFAULT 0 | 角色：0=普通用户，1=医生，2=管理员 |
| status | tinyint | — | NOT NULL, DEFAULT 0 | 账户状态：0=正常，1=禁用 |
| created_at | datetime | — | NOT NULL, DEFAULT CURRENT_TIMESTAMP | 注册时间 |

> **备注**：`role`字段预留角色扩展空间，未来可支持更多权限层级。

### 表2：设备表 (devices)

| 字段名 | 数据类型 | 主外键约束 | 非空规则 | 业务说明 |
|--------|----------|------------|-----------|---------|
| device_id | varchar(32) | PRIMARY KEY | NOT NULL | 设备唯一序列号，真实设备为出厂码，虚拟设备格式为VIR+16位随机字符 |
| user_id | bigint | FOREIGN KEY (users.user_id), NULL | NULL | 当前绑定用户ID，NULL表示未绑定 |
| device_name | varchar(50) | — | NOT NULL, DEFAULT '我的设备' | 用户自定义昵称 |
| is_virtual | tinyint | — | NOT NULL, DEFAULT 0 | 是否为虚拟设备：1=是，0=否 |
| firmware_version | varchar(20) | — | NOT NULL, DEFAULT 'V1.0.0' | 固件版本号 |
| last_active_time | datetime | — | NOT NULL, DEFAULT CURRENT_TIMESTAMP | 最后活跃时间 |
| created_at | datetime | — | NOT NULL, DEFAULT CURRENT_TIMESTAMP | 添加时间 |

> **备注**：`is_virtual`字段用于区分真实设备与虚拟设备，便于开发测试与生产环境共存。

### 表3：睡眠报告表 (sleep_reports)

| 字段名 | 数据类型 | 主外键约束 | 非空规则 | 业务说明 |
|--------|----------|------------|-----------|---------|
| report_id | bigint | PRIMARY KEY, AUTO_INCREMENT | NOT NULL | 报告唯一标识 |
| user_id | bigint | FOREIGN KEY (users.user_id), NOT NULL | NOT NULL | 所属用户ID |
| device_id | varchar(32) | FOREIGN KEY (devices.device_id), NOT NULL | NOT NULL | 产生报告的设备ID |
| report_date | date | — | NOT NULL | 报告对应夜晚日期，与user_id组成唯一索引 |
| sleep_score | int | — | NOT NULL, DEFAULT 0 | 睡眠综合评分（0–100） |
| total_minutes | int | — | NOT NULL, DEFAULT 0 | 总睡眠时长（分钟） |
| deep_minutes | int | — | NOT NULL, DEFAULT 0 | 深睡时长 |
| rem_minutes | int | — | NOT NULL, DEFAULT 0 | REM时长 |
| light_minutes | int | — | NOT NULL, DEFAULT 0 | 浅睡时长 |
| wake_minutes | int | — | NOT NULL, DEFAULT 0 | 夜间清醒总时长 |
| avg_heart_rate | decimal(5,2) | — | NULL | 整夜平均心率 |
| events_json | text | — | NULL | 异常事件列表（JSON数组） |
| heart_rate_curve | text | — | NULL | 分钟级心率数据（JSON数组） |
| respiration_curve | text | — | NULL | 分钟级呼吸频率数据 |
| stage_curve | text | — | NULL | 睡眠分期序列（每30秒一个值） |
| noise_curve | text | — | NULL | 环境噪音等级曲线（模拟） |
| created_at | datetime | — | NOT NULL, DEFAULT CURRENT_TIMESTAMP | 报告生成时间 |

> **备注**：所有曲线类字段使用JSON格式存储，便于前端解析与图表渲染，未来可考虑时序数据库优化。

### 表4：睡眠日记表 (sleep_diary)

| 字段名 | 数据类型 | 主外键约束 | 非空规则 | 业务说明 |
|--------|----------|------------|-----------|---------|
| id | bigint | PRIMARY KEY, AUTO_INCREMENT | NOT NULL | 日记条目唯一标识 |
| user_id | bigint | FOREIGN KEY (users.user_id), NOT NULL | NOT NULL | 所属用户ID |
| date | date | — | NOT NULL | 日记记录日期（对应起床日） |
| bed_time | time | — | NOT NULL | 上床时间 |
| sleep_latency_min | int | — | NOT NULL, DEFAULT 0 | 入睡潜伏期（分钟） |
| wakeup_times | int | — | NOT NULL, DEFAULT 0 | 夜间觉醒次数 |
| wakeup_time | time | — | NOT NULL | 起床时间 |
| daytime_sleepiness | int | — | NOT NULL | 日间困倦程度（1–5分） |
| notes | text | — | NULL | 自由备注 |
| created_at | datetime | — | NOT NULL, DEFAULT CURRENT_TIMESTAMP | 创建时间 |

> **备注**：此表为e-CBTi模块提供主观数据输入，支持长期趋势分析与治疗依从性评估。

### 表5：量表结果表 (questionnaire_results)

| 字段名 | 数据类型 | 主外键约束 | 非空规则 | 业务说明 |
|--------|----------|------------|-----------|---------|
| id | bigint | PRIMARY KEY, AUTO_INCREMENT | NOT NULL | 测评记录唯一标识 |
| user_id | bigint | FOREIGN KEY (users.user_id), NOT NULL | NOT NULL | 所属用户ID |
| type | varchar(10) | — | NOT NULL | 量表类型：ISI / PSQI / ESS |
| answers_json | text | — | NOT NULL | 各题答案（JSON数组） |
| total_score | int | — | NOT NULL | 总分（ISI:0–28, PSQI:0–21, ESS:0–24） |
| level | varchar(20) | — | NULL | 等级评定结果 |
| created_at | datetime | — | NOT NULL, DEFAULT CURRENT_TIMESTAMP | 提交时间 |

> **备注**：`answers_json`字段保留原始答题记录，支持未来回溯分析与算法优化；`level`字段由后端根据临床标准自动填充，确保一致性。
