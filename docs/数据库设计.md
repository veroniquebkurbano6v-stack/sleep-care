# 睡眠评估干预系统 - 数据库设计

本系统采用MySQL数据库，设计5张核心数据表，遵循规范化原则，确保数据一致性与查询效率。

## 关系说明

| 关系 | 类型 | 说明 |
|------|------|------|
| users → devices | 1:N | 一个用户可绑定多个设备 |
| users → sleep_reports | 1:N | 一个用户可有多个睡眠报告 |
| users → sleep_diary | 1:N | 一个用户可有多条睡眠日记 |
| users → questionnaire_results | 1:N | 一个用户可有多个量表测评结果 |
| users → user_settings | 1:1 | 一个用户对应一套设置 |
| devices → sleep_reports | 1:N | 一个设备可产生多个睡眠报告 |
| users(医生) → doctor_authorizations | 1:N | 一个医生可授权查看多个患者 |
| users(患者) → doctor_authorizations | 1:N | 一个患者可被多个医生授权查看 |

## 完整DDL

### 创建数据库

```sql
CREATE DATABASE IF NOT EXISTS sleep_care CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
USE sleep_care;
```

### 表1：用户表 (users)

```sql
CREATE TABLE IF NOT EXISTS users (
    user_id BIGINT PRIMARY KEY AUTO_INCREMENT COMMENT '用户唯一标识，系统自动生成',
    phone VARCHAR(20) NOT NULL UNIQUE COMMENT '手机号码，用于登录和身份识别',
    nickname VARCHAR(50) NOT NULL DEFAULT '用户' COMMENT '用户昵称，最长50字符',
    avatar_url VARCHAR(255) NULL COMMENT '头像URL地址',
    gender TINYINT NOT NULL DEFAULT 0 COMMENT '性别：0=未知，1=男，2=女',
    birth_year INT NULL COMMENT '出生年份，用于年龄计算',
    role TINYINT NOT NULL DEFAULT 0 COMMENT '角色：0=普通用户，1=医生，2=管理员',
    status TINYINT NOT NULL DEFAULT 0 COMMENT '账户状态：0=正常，1=禁用',
    created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '注册时间',
    INDEX idx_users_phone (phone),
    INDEX idx_users_role (role),
    INDEX idx_users_status (status)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='用户表';
```

### 表2：设备表 (devices)

```sql
CREATE TABLE IF NOT EXISTS devices (
    device_id VARCHAR(32) PRIMARY KEY COMMENT '设备唯一序列号，真实设备为出厂码，虚拟设备格式为VIR+16位随机字符',
    user_id BIGINT NULL COMMENT '当前绑定用户ID，NULL表示未绑定',
    device_name VARCHAR(50) NOT NULL DEFAULT '我的设备' COMMENT '用户自定义昵称',
    is_virtual TINYINT NOT NULL DEFAULT 0 COMMENT '是否为虚拟设备：1=是，0=否',
    firmware_version VARCHAR(20) NOT NULL DEFAULT 'V1.0.0' COMMENT '固件版本号',
    last_active_time DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '最后活跃时间',
    created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '添加时间',
    INDEX idx_devices_user_id (user_id),
    INDEX idx_devices_is_virtual (is_virtual),
    INDEX idx_devices_last_active (last_active_time),
    FOREIGN KEY (user_id) REFERENCES users(user_id) ON DELETE SET NULL
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='设备表';
```

### 表3：睡眠报告表 (sleep_reports)

```sql
CREATE TABLE IF NOT EXISTS sleep_reports (
    report_id BIGINT PRIMARY KEY AUTO_INCREMENT COMMENT '报告唯一标识',
    user_id BIGINT NOT NULL COMMENT '所属用户ID',
    device_id VARCHAR(32) NOT NULL COMMENT '产生报告的设备ID',
    report_date DATE NOT NULL COMMENT '报告对应夜晚日期',
    sleep_score INT NOT NULL DEFAULT 0 COMMENT '睡眠综合评分（0–100）',
    total_minutes INT NOT NULL DEFAULT 0 COMMENT '总睡眠时长（分钟）',
    deep_minutes INT NOT NULL DEFAULT 0 COMMENT '深睡时长',
    rem_minutes INT NOT NULL DEFAULT 0 COMMENT 'REM时长',
    light_minutes INT NOT NULL DEFAULT 0 COMMENT '浅睡时长',
    wake_minutes INT NOT NULL DEFAULT 0 COMMENT '夜间清醒总时长',
    avg_heart_rate DECIMAL(5,2) NULL COMMENT '整夜平均心率',
    events_json TEXT NULL COMMENT '异常事件列表（JSON数组）',
    heart_rate_curve TEXT NULL COMMENT '分钟级心率数据（JSON数组）',
    respiration_curve TEXT NULL COMMENT '分钟级呼吸频率数据',
    stage_curve TEXT NULL COMMENT '睡眠分期序列（每30秒一个值）',
    noise_curve TEXT NULL COMMENT '环境噪音等级曲线（模拟）',
    created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '报告生成时间',
    UNIQUE KEY uk_sleep_reports_user_date (user_id, report_date),
    INDEX idx_sleep_reports_device_id (device_id),
    INDEX idx_sleep_reports_report_date (report_date),
    FOREIGN KEY (user_id) REFERENCES users(user_id) ON DELETE CASCADE,
    FOREIGN KEY (device_id) REFERENCES devices(device_id) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='睡眠报告表';
```

### 表4：用户设置表 (user_settings)

```sql
CREATE TABLE IF NOT EXISTS user_settings (
    id BIGINT PRIMARY KEY AUTO_INCREMENT COMMENT '设置唯一标识',
    user_id BIGINT NOT NULL UNIQUE COMMENT '所属用户ID',
    bedtime TIME NULL COMMENT '预设就寝时间',
    wakeup_time TIME NULL COMMENT '预设起床时间',
    sunset_duration INT NOT NULL DEFAULT 10 COMMENT '日落程序时长（分钟）',
    sunrise_duration INT NOT NULL DEFAULT 10 COMMENT '日出程序时长（分钟）',
    light_temp_min INT NOT NULL DEFAULT 2200 COMMENT '最低色温（K）',
    light_temp_max INT NOT NULL DEFAULT 4000 COMMENT '最高色温（K）',
    sound_enabled TINYINT NOT NULL DEFAULT 1 COMMENT '是否启用助眠音效',
    sound_type VARCHAR(20) NOT NULL DEFAULT 'white' COMMENT '默认音效类型',
    noise_canceling TINYINT NOT NULL DEFAULT 0 COMMENT '是否启用主动降噪',
    notification_enabled TINYINT NOT NULL DEFAULT 1 COMMENT '是否启用消息推送',
    created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    updated_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    FOREIGN KEY (user_id) REFERENCES users(user_id) ON DELETE CASCADE

```

### 表5：睡眠日记表 (sleep_diary)

```sql
CREATE TABLE IF NOT EXISTS sleep_diary (
    id BIGINT PRIMARY KEY AUTO_INCREMENT COMMENT '日记条目唯一标识',

    sleep_latency_min INT NOT NULL DEFAULT 0 COMMENT '入睡潜伏期（分钟）',

    created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
```

### 表6：量表结果表 (questionnaire_results)

```sql
CREATE TABLE IF NOT EXISTS questionnaire_results (

    user_id BIGINT NOT NULL COMMENT '所属用户ID',

    answers_json TEXT NOT NULL COMMENT '各题答案（JSON数组）',

```

### 表7：医生授权表 (doctor_authorizations)

```sql
CREATE TABLE IF NOT EXISTS doctor_authorizations (
    id BIGINT PRIMARY KEY AUTO_INCREMENT COMMENT '授权记录唯一标识',
    doctor_id BIGINT NOT NULL COMMENT '医生用户ID',
    patient_id BIGINT NOT NULL COMMENT '患者用户ID',
    status TINYINT NOT NULL DEFAULT 1 COMMENT '授权状态：0=已取消，1=有效',
    created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',

    INDEX idx_doctor_authorizations_patient (patient_id),
    INDEX idx_doctor_authorizations_status (status),
    FOREIGN KEY (doctor_id) REFERENCES users(user_id) ON DELETE CASCADE,
    FOREIGN KEY (patient_id) REFERENCES users(user_id) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='医生授权表';
```

## 索引优化建议

1. **users表**：`phone`字段建立唯一索引用于快速登录查询；`role`和`status`建立普通索引用于权限筛选
2. **devices表**：`user_id`建立索引用于查询用户绑定设备；`is_virtual`建立索引用于筛选虚拟设备
3. **sleep_reports表**：`(user_id, report_date)`建立唯一索引确保每日只生成一份报告；`device_id`和`report_date`建立索引用于按设备和日期查询
4. **user_settings表**：`user_id`建立唯一索引确保每个用户只有一套设置
5. **sleep_diary表**：`(user_id, date)`建立唯一索引确保每日只记录一条日记
6. **questionnaire_results表**：`user_id`和`type`建立索引用于查询用户特定量表历史
7. **doctor_authorizations表**：`(doctor_id, patient_id)`建立唯一索引防止重复授权；`doctor_id`和`patient_id`分别建立索引用于双向查询

## JSON字段结构说明

### events_json 结构

```json
[
    {
        "type": "apnea",
        "start_time": "01:23:45",
        "duration": 15,
        "level": "moderate"
    }
]
```

### heart_rate_curve 结构

```json
[65, 66, 64, 63, ...]
```

### stage_curve 结构

```json
[0, 0, 1, 1, 2, 2, 1, 3, ...]
```

**stage编码说明**：
- 0 = 清醒
- 1 = 浅睡
- 2 = 深睡
- 3 = REM

## 数据安全考虑

1. **脱敏处理**：医生查看患者数据时，手机号等敏感信息需脱敏显示
2. **权限控制**：通过`role`字段和`doctor_authorizations`表实现数据访问控制
3. **日志记录**：关键操作（如授权变更、数据导出）需记录操作日志
4. **备份策略**：定期全量备份+增量备份，确保数据可恢复
